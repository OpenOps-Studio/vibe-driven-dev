# Vibe Driven Dev Compatibility Layer

This project installs VDD as a canonical workflow layer and exposes it through a portable `AGENTS.md` contract.

## Role and Operating Mode

You are operating inside a VDD-managed project. Treat the VDD workspace as the source of truth for execution flow, delivery stages, and handoff quality.

## Canonical Agent Exports

The following VDD agent contracts are exported into `.vdd/agents/`:

- `.vdd/agents/vdd-orchestrator.md`
- `.vdd/agents/vdd-planner.md`
- `.vdd/agents/vdd-architect.md`
- `.vdd/agents/vdd-detailer.md`
- `.vdd/agents/vdd-researcher.md`
- `.vdd/agents/vdd-qa-guardian.md`
- `.vdd/agents/vdd-handoff-manager.md`

## Working Rules

1. Start from the VDD project state in `.vdd/project-state.json` when it exists.
2. Prefer the canonical VDD agents in `.vdd/agents/` over ad hoc workflows.
3. Use `vdd scan`, `vdd status`, and `vdd validate <sourceId>` to inspect project runtime state.
4. Treat installed packs in `.vdd/packs.json` as executable extensions only after validation.
5. Keep changes minimal, testable, and aligned to the current project stage.
