# OpenOps Studio Publish Spec

## Goal

Prepare `vibe-driven-dev` for a first public release under `OpenOps-Studio` with a clearer open-source identity, stronger first impression, and cleaner contributor onboarding.

## Success criteria

- the repository name remains `vibe-driven-dev`
- the README explains the product, audience, workflow, and quick start within one screen
- public-facing docs exist for contribution, conduct, and security reporting
